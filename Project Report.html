Goal of the project¶

The goal of the project is to identify the persons of interest in Enron scandal if a new dataset is given.We are aiming to build a POI idenfier model using the current dataset by Supervised learning. Depending on how we choose the features and amount of data available, Machine learning gives us a predictive model which can be used on other datasets with certain level of accuracy.

The information available in the dataset is the different types of payment given to the insiders, email details, & POI identifier for a list of insiders.

Machine Learning helps us identify patterns in the data and relevance of the selected features in predicting the outcome and brings out hidden insights in the data. There are various algorithms avalaible in Machine learning and which one to use depends on the dataset given and the amount of data.

As mentioned earlier there are financial as well as email information for different insiders. Understanding the different features available is the first task of Machine Learning.

Feature Selection¶

Some of the features given are modified to better represent the scenario like to_messages_ratio & from_messages_ratio which gives a ratio of emails sent to POIs or from POIs in the total amount of resepctive emails (from or to). Also another feature is added which is IsDirector to represent whether a particular insider is a director or not depending on whether that particular insider is given director fees or not. This has been done on the hypothesis that directors have more chance to be a POI as they have higher influence in taking company decissions. As we are taking IsDirector feature, I opted to remove director_fees feature as it kind of becomes redundant. Nothing much can be drawn from resticted stock features or deferal payments and are opted to be left in the features list. SelectKbest is then used on the new set features to filter out 10 best features.

IsDirector feature that we have selected seems to be a good feature from SelectKbest scores.

Feature Scaling¶

Feature Scaling is not used in the prediction, as the alogorithms used are NaiveBayes & Decision Trees

Algorithms Used¶

NaiveBayes & Decision Tree are chosen for the prediction. NaiveBayes algorithm by its nature does not require any parameter input. As Decision trees often tend to be overfitting if not optimized properly, we need chose our parameters accordingly.

Parameter Tuning¶

Naive Bayes does not require any parameters and hence there is no need for parameter tuning in this case.

Decision trees use various parameters some which can be optimized for better performance and to avoid overfitting. The parameters chosen to be optimized for the decission tree classifier are min_samples_leaf and min_samples_split. Min_samples_leaf helps to minimize overfitting.Min_samples_split is chosen to be optimized as we have seen from the exploratory analsyis that there are no clear boundaries between the classes and we might require more splits than the default value 2. Given below is the perfomance when different sets of these parameters chosen for prediction. GridSearchCV is not used for this task as apparently there sklearn package installed (v 1.8) in my system does not have this module. So I have used manual selection of the parameters. Parameter settings used are as given below: {'min_samples_split':[2,3,4]} {'min_samples_leaf':[1, 2,3,4,5]}

But when opted to use different parameter values than the default values for min_samples_split and min_samples_leaf, the recall value drops down by almost 50%.

Validation¶

After the model is trained on a part of the dataset given, it needs to be tested on the other part as a validation to see if the model is giving required level of accuracy, precision and recall

Depending on the algorithm we choose, we need to select a suitable set of parameter values for the algorithm which is not always straight forward. In such cases after dividing the dataset into train and test sets, we further divide the train set into train set and validation set so that parameter selection can be tested on the validation set. Once the best parameter setting is chosen, the model is tested on the test set. If the test set is used in selecting the parameters and again later the same test set is used to test the model performance, the model may not be able to generalise to unseen data in this case.

Evalulation Metrics¶

The dataset is divided into two datasets with 70% and 30% of data respectively. The model is being trained on the 70% of the data and tested on the other 30%
